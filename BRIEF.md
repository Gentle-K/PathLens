# Genius Actuary 项目简介

## 一、项目定位

Genius Actuary 现在的主定位不是“通用个人决策助手”，而是 **HashKey Chain 上的 RWA 可验证决策与执行层**。

它的核心目标不是只给出一份分析报告，而是围绕链上真实资产回答 5 个问题：

- 这个资产是否真实、依据是什么
- 这个钱包是否具备准入资格
- 这个资产是链上直连、发行方入口，还是仅可查看
- 赎回与流动性窗口是什么
- 买入之后应该监控哪些证明、价格、KYC 和发行方风险信号

AI 分析仍然保留，但现在属于入口能力；真正的主产品是一个围绕 **资产证明 -> 执行 -> 监控** 展开的 RWA Hub。

## 二、这个项目能解决什么问题

当前 RWA / DeFi 产品常见的问题不是“没有建议”，而是：

- 资产披露和链上状态分散，用户很难快速判断真实性
- 一个资产看起来能买，但实际上还要走发行方或白名单流程
- demo 资产、benchmark 资产和真实可执行资产经常被混在一起
- 买入之后缺少 proof freshness、oracle、KYC、赎回窗口等持续监控

Genius Actuary 要解决的是把这些关键判断前置并结构化：

- 用 proof snapshot 把资产来源、合约、披露和 freshness 汇总到一张卡片
- 用 readiness 把 direct contract、issuer portal、view only 明确区分
- 用组合监控把 proof、oracle、KYC、issuer workflow 告警持续显示
- 用分析报告解释“为什么推荐”，而不是替代资产真实性验证

## 三、适用场景

Genius Actuary 现在聚焦于 HashKey 单链 RWA 场景，例如：

- 稳定币与 MMF 的链上现金管理选择
- 贵金属 RWA 与稳定币/MMF 的风险收益对比
- 专业投资者可准入资产的资格判断与执行路径识别
- 发行方入口资产的赎回节奏、托管与 proof 监控

这类场景的共同点是：

- 既需要金融分析，也需要链上/披露层验证
- 资产可买性并不等于链上可直接执行
- 投后监控与事前判断同样重要

## 四、核心能力

### 1. 结构化拆解问题

当用户输入一个问题后，系统不会直接仓促回答，而是先理解这是一个什么类型的决策，再把它拆成更适合分析的几个维度，例如：

- 目标是什么
- 约束条件是什么
- 可能的成本有哪些
- 主要风险在哪里
- 有哪些可以比较的方案

这一步可以帮助用户从“感觉上很复杂”走向“知道该看哪些重点”。

### 2. 主动补充关键信息

很多决策之所以难，是因为关键信息并不完整。

Genius Actuary 会像一个有经验的顾问一样，继续向用户追问必要信息，例如预算、时间安排、风险偏好、个人目标、现实限制等。通过这种方式，系统不是被动等用户一次性说完，而是主动帮助用户把问题讲清楚。

### 3. 成本与风险分析

系统会从多个角度帮助用户识别隐藏成本与潜在风险，例如：

- 金钱成本
- 时间成本
- 机会成本
- 执行难度
- 短期风险
- 长期风险
- 外部环境变化带来的影响

相比简单建议，这种分析更适合真正重要的决策。

### 4. 多方案对比

当用户不是在问“要不要做”，而是在问“到底选哪一个”时，系统可以把多个选项放在同一个分析框架里进行比较，帮助用户更直观地看清差异。

### 5. 生成结构化报告

分析完成后，系统会输出一份清晰的结果说明，而不是零散对话记录。报告通常会包括：

- 问题定义
- 已知条件
- 成本与风险分析
- 方案比较
- 当前结论
- 建议行动方向
- 尚未确定的关键点

## 五、用户如何使用

整个使用流程非常自然，适合普通用户直接上手：

1. 用户输入一个真实问题
2. 系统快速识别问题类型并提出关键补充问题
3. 用户逐步补充背景信息
4. 系统持续整理和分析内容
5. 输出结构化分析结果与建议

这个过程和普通问答最大的不同是：

- 它更重视“把问题问完整”
- 它更重视“分析过程”
- 它更重视“结论为什么成立”

## 六、对客户的价值

Genius Actuary 的价值不只是“回答问题”，而是提升用户面对重要决策时的判断质量。

对于客户而言，它可以带来的价值包括：

- 提高用户决策的条理性
- 减少因为信息不完整而导致的误判
- 让复杂问题变得更容易理解与比较
- 增强用户对分析结果的信任感
- 形成可复盘、可展示、可沉淀的输出结果

如果未来用于更具体的垂直方向，它还可以进一步发展成特定场景下的决策助手，例如：

- 留学与升学决策助手
- 职业发展决策助手
- 副业与创业项目评估助手
- 高价值消费决策助手

## 七、项目输出形式

这个项目最终呈现为一个可交互的 Web 产品。

用户不需要掌握复杂操作，只需要像平时提问一样输入问题，就可以逐步完成一轮完整分析。系统最终输出的不是一段模糊回答，而是一份更适合阅读、比较和理解的分析结果。

它既可以作为独立产品使用，也具备进一步包装成行业化解决方案的潜力。

## 八、项目总结

Genius Actuary 的核心意义，在于把“复杂决策”从模糊、主观、零散的状态，转变为一个可分析、可解释、可输出的过程。

它面向的不是随意闲聊，而是真正重要的问题。

它提供的不是一句答案，而是一套更接近专业顾问思路的分析体验。

如果客户希望打造一款能够体现“AI 真正帮助用户思考”的产品，Genius Actuary 是一个非常适合继续打磨与落地的方向。
