# 比赛 Demo 视频讲稿

## 推荐时长

- 主视频：90 到 120 秒
- 可选补充：额外插入 30 到 45 秒的 REST 实录片段

## 主视频镜头顺序

### 0:00 - 0:08 登录页

画面：

- 展示 `Genius Actuary` 登录页
- 点击 `Open demo workspace`

旁白：

- 这是 `Genius Actuary`，一个面向 HashKey RWA 的可验证决策、执行与监控工作台。

### 0:08 - 0:20 新建分析

画面：

- 进入 `/new-analysis`
- 输入一条资金配置需求
- 点击 `Create session`

旁白：

- 用户只需要输入一条资金目标，系统就会把它转换成结构化的 RWA 分析任务。

### 0:20 - 0:33 澄清问题

画面：

- 展示澄清页
- 快速完成问题
- 点击 `Continue analysis`

旁白：

- 在进入分析前，系统会先补齐流动性、风险偏好和约束条件，避免后续建议失真。

### 0:33 - 0:50 报告生成

画面：

- 展示分析进行中页面
- 自动跳转到报告页
- 缓慢滚动报告内容

旁白：

- 随后系统会生成完整报告，包含资产排序、收益与风险模拟、证据链，以及可执行的下一步动作。

### 0:50 - 1:02 Proof 页面

画面：

- 切到 `/assets/hsk-usdc/proof`
- 展示 proof freshness、timeline 和 anchor 状态

旁白：

- 每一个资产都有独立的 proof 页面，用来验证披露信息、时间线和链上锚定状态。

### 1:02 - 1:18 Execution 页面

画面：

- 切到 `/sessions/{sessionId}/execute`
- 点击 `Generate submit receipt`
- 展示 receipt 弹层

旁白：

- 从报告可以直接进入执行控制台，先生成 receipt，再决定是否写入真实交易数据。

### 1:18 - 1:30 Portfolio 页面

画面：

- 切到 `/portfolio/{address}`
- 展示持仓与提醒

旁白：

- 执行之后，监控页会统一展示持仓、proof freshness 和后续告警。

### 1:30 - 1:40 Debug / Ops 页面

画面：

- 切到 `/debug/rwa-ops`

旁白：

- 对运营和评审方来说，我们还提供独立的运维视图，用于 proof 刷新、索引和状态核查。

### 1:40 - 1:50 结尾页

画面：

- 回到报告页
- 停在结尾字幕

旁白：

- 这个 demo 路径在 `mock/demo` 模式下稳定可复现，同时项目也保留了一条 REST-backed 的真实后端闭环来证明执行能力。

## 可选 REST 补充片段

建议插在 `Execution 页面` 后面。

旁白：

- 除了稳定的比赛演示模式，我们还保留了一条真实后端链路，可以从报告进入执行、记录交易哈希、写入 anchor，再回到组合监控页面完成闭环。

## 录制建议

- 用英文界面录制，中文配音最稳妥
- 不要超过两分钟
- 全程只讲三件事：`decision`、`proof`、`execution + monitoring`
- 如果比赛时间很紧，只交主视频即可
